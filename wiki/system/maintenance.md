# Maintenance

## Weekly

- [ ] **Check for orphaned notes** — notes with no incoming links in your vault
- [ ] **Review MEMORY.md** — delete entries that are no longer relevant
- [ ] **Promote stable entries** — if MEMORY.md exceeds 4000 chars, move stable entries to vault files

## Monthly

- [ ] **Audit vault structure** — are files in the right folders?
- [ ] **Check System/Assistant/** — update outdated information in context.md, preferences.md, environment.md
- [ ] **Review Issues & Fixes Log** — look for recurring patterns worth documenting as skills

## When Something Breaks

1. **Log it** — write the issue in `System/Assistant/logs/issues-fixes-log.md`
2. **Save the fix** — if it's a recurring workflow, save it as a skill/procedure file
3. **Update environment.md** — if it reveals a new known issue pattern

## Principles

- Plain text = no vendor lock-in
- The vault must be fully functional without the AI
- Skills > manual fixes — automate recurring patterns
- If it broke twice, document it thrice
